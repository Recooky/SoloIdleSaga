一、项目全部 BUG & 严重问题汇总（分类整理，按优先级从高→低）
🔴 致命级（直接白屏、函数未定义、功能完全不可用，必须最先修复）
1. JS 脚本加载顺序错乱 + 重复引入脚本
index.html 多处脚本顺序颠倒：versionUpdate.js 在 main.js 之前加载，window.GAME_VERSION 未定义报错；cloudSave.js 放在 app.js 之后，云存档全局方法先被调用还未加载；页面末尾重复引入 main.js，导致初始化逻辑执行 2 次。
battle.html、character.html 使用 type="module" 引入页面脚本，模块隔离无法访问window上所有全局配置 / 工具函数，大量xxx is not defined。
三个子页面缺失完整依赖脚本引入，只引入了页面业务 JS，缺少前置配置、核心工具类脚本。
2. 全局函数重复定义冲突
main.js、app.js 同时定义 window.getRarityClass，后加载的文件会覆盖前一个，极易出现装备稀有度样式渲染异常。
3. 战斗回调函数重复绑定
app.js 中连续两次调用 initBattleCallback 注册刷新 UI 回调：
后果：每次战斗日志、血量、掉落更新，页面刷新逻辑执行 2 次，界面闪烁、日志重复、装备重复渲染。
4. 多页面 DOM 缺失关键元素
battle.html 缺少「开始挂机、停止挂机」按钮 DOM，无法启停战斗。
battle.html、character.html 缺失：登录弹窗、账号下拉菜单、云端存档时间展示 DOM，云登录、云存档功能直接失效。
🟠 严重级（功能可用但逻辑异常、数据错乱、体验崩坏）
5. 装备掉落概率写死 100% 必掉
equipGenerator.js 的 monsterDropEquip 中普通 / 精英 / BOSS 怪物掉落概率全部 = 1，每只怪物必掉落装备，短时间背包堆满，触发localStorage存储上限、存档损坏。
6. 离线收益同样 100% 必掉装备
离线挂机结算调用同样是monsterDropEquip，会批量生成大量装备塞满背包。
7. 全局变量未用let/const声明，隐式挂载全局污染
app.js 中 currentTab、battleTimer、bagRefreshTimer、batchFilterCache 未声明，变成隐式全局变量，容易出现变量被意外覆盖、逻辑错乱。
8. 页面粗暴全局刷新方式存在缺陷
refreshCharacterPanel 通过清空mainWrap的innerHTML再赋值实现刷新：
动态弹窗、事件委托以外的点击事件全部失效、输入框状态丢失、界面闪烁。
🟡 高危隐性 BUG（边界场景触发，后期容易崩档）
9. 战斗存在重复开启定时器漏洞
玩家在怪物死亡 300ms 延迟区间重复点击【开始战斗】，会创建多组setInterval定时器，出现双倍攻击、日志疯狂刷屏、装备疯狂掉落。
10. 背包无容量上限限制
长期离线挂机、反复刷 BOSS，装备数量无限膨胀，localStorage、云端数据库单条存档会超出存储限制，导致存档上传失败、本地存档损坏丢失。
11. 批量销毁装备无二次确认弹窗
点击确定销毁直接删除装备，误操作无法回滚，玩家装备被误删会直接损失进度。
12. 页面关闭beforeunload事件执行不可靠
浏览器强制关闭标签页时，存在概率无法执行停止战斗、更新离线时间逻辑，导致离线收益结算异常、战斗定时器残留。
🟢 优化类问题（不报错，但体验差、可维护性低）
13. 版本更新检测是异步函数，但初始化没有await串行等待
main.js 中checkVersionUpdate()异步版本校验没有 await，可能旧代码先初始化再触发页面刷新，出现短暂异常展示。
14. 游客模式缺失，强制必须登录才能游玩
打开页面直接弹出登录弹窗，无法纯本地离线游玩，不想用云存档的玩家无法体验游戏。
15. 装备稀有度两处映射表冗余维护
main.js、app.js 各自写了一套稀有度映射配置，后续新增稀有度需要两处同时修改，极易漏改出 BUG。
